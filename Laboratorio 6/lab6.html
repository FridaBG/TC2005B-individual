questions

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

answers

  1. Es una buena práctica usar JavaScript para validar los inputs de los formularios antes de enviar los datos al servidor por las siguientes razones:
    • Proporciona una retroalimentación inmediata al usuario sobre cualquier error en los datos ingresados, lo que mejora la experiencia del usuario.
    • Ayuda a evitar que se envíen datos incorrectos al servidor, lo que puede generar problemas de seguridad o errores en la aplicación.
    • Puede reducir las solicitudes innecesarias al servidor, lo que mejora la eficiencia y la velocidad del sitio web.
  2. Es posible saltarse la seguridad de las validaciones hechas con JavaScript mediante la manipulación del código fuente de la página web o mediante el uso de herramientas especializadas, como los proxies web. Sin embargo, estas técnicas pueden ser muy complicadas de implementar y, en la mayoría de los casos, solo se utilizan con fines maliciosos.
  3. Aunque es posible saltarse la seguridad de las validaciones hechas con JavaScript, sigue siendo una buena práctica utilizar JavaScript para validar los inputs de los formularios antes de enviar los datos al servidor porque:
    • Proporciona una capa adicional de seguridad y evita que se envíen datos incorrectos al servidor.
    • Mejora la experiencia del usuario al proporcionar retroalimentación inmediata sobre cualquier error en los datos ingresados.
    • Puede reducir las solicitudes innecesarias al servidor, lo que mejora la eficiencia y la velocidad del sitio web.
