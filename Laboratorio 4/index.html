  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: solicita una representación de un recurso específico, recuperar datos.
    • HEAD: pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
    • POST: se utiliza para enviar una entidad a un recurso en específico. Cambia el servidor.
    • PUT: reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
    • PATCH: aplica modificaciones parciales a un recurso.
    • DELETE: borra un recurso en específico.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    El error 404 se produce cuando un usuario intenta acceder a una página que no existe o no está disponible en el servidor. Por lo que si es responsabilidad del desarrollador corregirlo, pues esto puede ser causado por imagenes o archivos eliminados, que haya problemas de conexión o que el nombre del dominio haya cambiado.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    significa que su uso no es recomendado o se espera que sea reemplazado por una alternativa más moderna.

    • La etiqueta "frame": en HTML4, se utilizaba la etiqueta "frame" para crear un marco dentro de una página web. En HTML5, esta etiqueta ha sido desaprobada y se recomienda utilizar "iframe".
    • Atributos de presentación: en HTML4, los atributos de presentación, como "align", "bgcolor" y "width", se utilizaban para controlar el aspecto de los elementos de una página. En HTML5, estos atributos están desaprobados y se recomienda utilizar CSS.
    • Atributo "target": en HTML4, el atributo "target" se utilizaba para controlar dónde se abría un enlace. En HTML5, este atributo ha sido desaprobado y se recomienda utilizar "rel".
    • La etiqueta "font": en HTML4, la etiqueta "font" se utilizaba para controlar la apariencia de la fuente. En HTML5, esta etiqueta ha sido desaprobada y se recomienda utilizar CSS en su lugar.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    1. Semántica: HTML5 introduce nuevos elementos y atributos con un enfoque en la semántica, lo que permite a los desarrolladores describir de manera más precisa y accesible el contenido y la estructura de una página.
    2. Multimedia: HTML5 permite la incorporación de elementos multimedia como audio, video y animaciones sin la necesidad de plugins externos, como Flash.
    3. Formularios: HTML5 agrega nuevos tipos de controles para formularios, como fechas, números y correos electrónicos, que mejoran la experiencia de usuario y simplifican el procesamiento de formularios por parte del servidor.
    4. Aplicaciones en línea: HTML5 permite la creación de aplicaciones en línea sin necesidad de plugins externos, lo que mejora la compatibilidad, el rendimiento y la seguridad.
    5. Elementos depreciados: HTML5 desaprueba algunos elementos y atributos de HTML4 que ya no son necesarios o no se utilizan con frecuencia, como "frame", "font" y "target".
    6. Mejor rendimiento: HTML5 es más eficiente y rápido que HTML4, lo que mejora la experiencia de usuario y la accesibilidad de la página.

  14. Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

    El ciclo de vida de los sistemas de información es el proceso que describe las etapas por las que pasa un sistema de información, desde su concepción hasta su obsolescencia y retirada. Generalmente se compone de las siguientes etapas: planificación, análisis, diseño, implementación y mantenimiento. El ciclo de desarrollo de sistemas de información es un modelo que describe las fases que se siguen en la creación de un sistema de información, desde la identificación de las necesidades del usuario hasta la implementación del sistema. Los modelos más comunes son el modelo en cascada, el modelo en espiral y el modelo de desarrollo ágil.
