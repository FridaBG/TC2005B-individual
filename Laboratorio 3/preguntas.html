1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
  1. Que se utilice con precaución y sólo en casos en los que sea absolutamente necesario. El uso excesivo de !important puede dificultar la lectura y el mantenimiento del código, ya que puede ocultar errores y hacer que el código sea más difícil de entender y modificar. Siempre es preferible utilizar la cascada natural de CSS para establecer el orden de precedencia de las reglas de estilo y evitar el uso de !important a menos que sea absolutamente necesario.
  2. Es importante elegir cuidadosamente una imagen de fondo en una página HTML porque una imagen grande y pesada puede ralentizar significativamente el tiempo de carga de la página. Se debe seleccionar una imagen que tenga una resolución y tamaño adecuados para el uso previsto, y optimizarla para reducir su tamaño de archivo y garantizar que se cargue rápidamente.
  3. Utilizar unidades relativas como porcentajes o ems en lugar de unidades absolutas como píxeles o puntos, ya que las unidades relativas permiten que el diseño de la página sea más flexible y se ajuste automáticamente a diferentes tamaños de pantalla y dispositivos. Las unidades absolutas son útiles para propiedades como el tamaño de fuente o el espaciado, pero en general, es mejor utilizar unidades relativas siempre que sea posible.
  4. El uso de una versión minimizada de CSS puede mejorar el rendimiento de un sitio web al reducir el tamaño del archivo CSS y, por lo tanto, reducir el tiempo de carga de la página. La versión minimizada elimina espacios en blanco, saltos de línea y otros caracteres innecesarios del código, lo que hace que el archivo sea más compacto y rápido de descargar. Es importante tener en cuenta que aunque la versión minimizada puede mejorar el rendimiento, puede hacer que el código sea más difícil de leer y modificar, por lo que es importante mantener una copia del código sin minimizar para fines de mantenimiento.
